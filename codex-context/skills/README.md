# Skills do projeto

Coloque aqui skills personalizadas do projeto quando quiser que outra conta do Codex tenha o mesmo contexto operacional.

Formato sugerido:
- uma pasta por skill
- cada skill com um `SKILL.md`

Exemplo:
- `codex-context/skills/saas-extintores-copy/SKILL.md`
- `codex-context/skills/saas-extintores-dev/SKILL.md`

Use skills para:
- tom de copy e marketing do SaaS
- regras de produto e negocio
- padroes tecnicos do projeto
- fluxos recorrentes de suporte, deploy ou atendimento
